# Onboarding 设计闭环审查与补强方案

版本：v1.0  
日期：2026-07-02  
审查范围：产品方案、后台设计、整体架构、落地计划、UI 设计提示词

## 1. 审查结论

当前方案的主路径成立：每日任务、知识点学习、配套练习、错题沉淀、AI 答疑、掌握度追踪已经形成基础闭环。

需要补强的是“断点处理”和“回流机制”。也就是用户首次进入、任务未完成、AI 答疑后、错题重做后、订阅授权失败后，系统下一步应该怎么把用户带回学习节奏。

## 2. 已闭合的链路

### 2.1 日常学习链路

```text
进入今日页 -> 查看今日 15 个知识点 -> 学习知识卡 -> 做配套题 -> 更新进度
```

健康度：可落地。  
补强点：需要明确今日任务未完成时的延期规则。

### 2.2 错题链路

```text
答题错误 -> 记录错题 -> 按章节归类 -> 错题重做 -> 更新掌握度
```

健康度：基本闭合。  
补强点：不能只靠用户手动标记掌握，需要有“连续答对后自动降频”的规则。

### 2.3 AI 答疑链路

```text
用户提问 -> 知识库检索 -> AI 回答 -> 展示来源 -> 保存会话
```

健康度：可落地。  
补强点：AI 回答后应该能回流到知识点、错题或复习任务，否则 AI 会变成孤立聊天框。

## 3. 需要补强的 6 个闭环

| 闭环 | 当前缺口 | 补强方案 | 优先级 |
| --- | --- | --- | --- |
| 首次进入 | 登录后直接进首页，但用户可能没有考试日期和计划 | 增加首次引导：考试日期、每日目标、提醒时间、资料状态 | P0 |
| 内容来源 | 有知识点和题库表，但资料导入流程不完整 | 增加内容导入状态：资料上传、解析、入库、同步知识库 | P0 |
| 今日任务 | 未完成任务如何处理不明确 | 未完成项进入次日复习候选池，最多保留 3 天高优先级 | P0 |
| AI 回流 | AI 问答没有反向影响学习计划 | AI 问答可关联知识点，标记疑难，并加入复习队列 | P1 |
| 错题掌握 | 手动标记掌握不可靠 | 错题连续 2 次重做正确后自动标记掌握，掌握后降频 | P1 |
| 订阅提醒 | 用户拒绝授权后的兜底不足 | 首页保留任务状态，用户完成关键动作后再温和请求订阅 | P1 |

## 4. 补强后的完整产品闭环

```text
首次进入
-> 设置考试日期、每日目标、提醒时间
-> 系统生成今日任务
-> 学习知识点
-> 完成配套题
-> 正确则提升掌握度
-> 错误则进入错题本
-> 用户可问 AI
-> AI 回答关联知识点和来源
-> 疑难点进入复习队列
-> 次日任务根据新知识、错题、疑难点、未完成项重新排序
```

## 5. 页面补强

### 5.1 首次引导页

目的：让用户在 60 秒内完成学习计划初始化。

步骤：

1. 微信登录。
2. 设置考试日期。
3. 设置每日目标，默认 15 个知识点。
4. 设置提醒时间，默认 08:30。
5. 告知“可以先使用内置知识库，后续再导入资料”。
6. 进入今日页并生成第一天任务。

### 5.2 今日页补强

需要增加：

- 昨日未完成提示。
- 今天推荐来源说明。
- 下一步动作按钮：继续学习、重做错题、问 AI。
- 没有题目时的兜底：先学习知识点，并提示“该知识点题目待补充”。

### 5.3 AI 页补强

AI 回答后提供 3 个明确动作：

- 加入复习。
- 出题考我。
- 查看关联知识点。

MVP 中“出题考我”可以先跳到已有题目，不直接生成新题。

### 5.4 错题本补强

错题状态：

```text
待重做
已重做但未掌握
已掌握
```

掌握规则：

- 用户手动标记掌握：立即隐藏在默认错题列表，但保留记录。
- 连续 2 次重做正确：自动标记掌握。
- 标记掌握后 7 天进入低频抽查。

## 6. 后台补强

### 6.1 新增业务状态

每日计划项状态：

```text
PENDING
STUDYING
QUIZ_PENDING
DONE
SKIPPED
EXPIRED
```

错题状态：

```text
OPEN
RETRYING
MASTERED
ARCHIVED
```

AI 回流动作：

```text
ADD_TO_REVIEW
LINK_KNOWLEDGE_POINT
EXPLAIN_QUESTION
PRACTICE_RELATED
```

### 6.2 新增数据建议

- `user_onboarding_state`：记录首次引导完成情况。
- `content_import_job`：记录资料或题库导入进度。
- `ai_action_record`：记录 AI 回答后的用户动作。
- `review_queue`：统一承接错题、疑难点、未完成任务。

## 7. 落地顺序调整

建议先按下面顺序实现：

```text
M0 项目骨架
M1 首次引导 + 登录 + 用户学习设置
M2 内容种子数据 + 今日任务
M3 答题 + 错题 + 掌握度
M4 AI 问答 + AI 回流到复习队列
M5 订阅提醒 + 任务未完成兜底
M6 视觉动效和体验打磨
```

这样每一阶段都能形成可用闭环，而不是堆功能。

## 8. 验收补强

MVP 验收应增加以下条件：

- 新用户首次进入能完成计划初始化。
- 今日任务未完成后，次日能看到延期或复习提示。
- AI 回答后能加入复习队列或关联知识点。
- 错题连续重做正确后能自动标记掌握。
- 用户拒绝订阅后仍能在首页看到清晰的今日任务状态。
- 内容导入失败时能在后台看到失败原因，不影响已有学习任务。

## 9. 实现影响

当前已经开始的 M0 项目骨架可以继续，不需要返工。

需要在后续实现中新增：

- 首次引导页和用户学习设置接口。
- review queue 统一复习队列。
- 内容导入状态表。
- AI 回答后的动作记录。
- 未完成任务滚动规则。

这些补强不会改变技术选型，只会让产品闭环更稳。
