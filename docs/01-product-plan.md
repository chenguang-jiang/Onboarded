# Onboarding 产品方案

版本：v1.0  
日期：2026-07-01  
产品定位：软考系统架构师备考教练微信小程序

## 1. 产品概述

Onboarding 面向软考系统架构师备考场景，核心目标不是做一个资料仓库，而是做一个能持续驱动学习的备考教练。

产品通过“每日知识点推送、每点一题、AI 知识库问答、错题复盘、掌握度追踪”形成闭环，帮助用户减少每天规划复习内容的成本。

核心闭环：

```text
生成今日任务 -> 学知识点 -> 做对应题 -> 记录结果 -> 沉淀错题 -> AI 答疑 -> 调整后续复习计划
```

补强后的完整闭环：

```text
首次进入设置计划
-> 生成今日任务
-> 学习知识点
-> 做配套题
-> 正确则提升掌握度
-> 错误则进入错题本
-> AI 答疑关联知识点
-> 疑难点、错题、未完成项进入复习队列
-> 次日推荐重新排序
```

详细闭环审查见 `docs/06-design-closure-review.md`。

## 2. 目标用户

### 2.1 核心用户

- 正在准备软考系统架构设计师考试的个人用户。
- 已经有教材、笔记、真题资料，但缺少系统化复习节奏。
- 希望通过碎片时间持续复习，而不是一次性大块刷资料。

### 2.2 用户痛点

- 知识点多，章节跨度大，每天不知道复习什么。
- 看过的内容容易忘，缺少复习节奏。
- 做错题后没有形成有效回炉机制。
- 遇到概念问题时，普通搜索结果不够聚焦考试语境。
- 临考前很难判断自己的薄弱章节。

## 3. 产品目标

### 3.1 MVP 目标

- 用户每天打开小程序即可看到 15 个推荐知识点。
- 每个知识点绑定一道练习题，学习后立即练习。
- 答错题目自动进入错题本，并按章节归类。
- 用户可以通过 AI 基于知识库提问，并获得带来源的回答。
- 系统可以展示章节掌握度和今日学习进度。

### 3.2 长期目标

- 建立个人化复习节奏。
- 基于错题、问答、掌握度自动调整每日任务。
- 支持真题套卷、模拟考试、考前冲刺计划。
- 支持导入用户自己的 PDF、Markdown、Excel 笔记或题库。

## 4. 产品原则

1. 简洁优先：每天只回答“今天学什么、练什么、复习什么”。
2. 结果导向：每次学习都产生记录，记录反向影响推荐。
3. AI 有边界：AI 回答必须优先基于知识库，缺少依据时明确提示。
4. 少打扰：微信订阅消息只做每日任务提醒，不做高频轰炸。
5. 有趣但克制：交互动效提升完成感，不影响学习效率。

## 5. 信息架构

底部 Tab 建议设置 4 个：

```text
今日
题库
AI
我的
```

### 5.1 今日

首页是核心入口，突出“今日任务舱”。

主要模块：

- 今日完成进度。
- 15 个知识点卡片。
- 当前待练题目。
- 错题待复习提醒。
- 薄弱章节提示。
- AI 快问入口。

首页关键指标：

- 今日知识点完成数。
- 今日题目完成数。
- 正确率。
- 连续学习天数。
- 距离考试天数。

### 5.2 题库

主要模块：

- 按章节练习。
- 今日配套练习。
- 错题重做。
- 收藏题。
- 模拟测试，MVP 后置。
- 真题套卷，MVP 后置。

### 5.3 AI

主要模块：

- 知识库问答。
- 历史会话。
- 快捷追问。
- 题目解析。
- 生成变式题，MVP 后置。

AI 回答固定结构：

```text
结论
解释
考试记忆点
易错提醒
参考来源
推荐练习
```

### 5.4 我的

主要模块：

- 学习计划。
- 考试日期。
- 每日学习量。
- 章节掌握度。
- 错题本入口。
- 知识库管理，MVP 可仅后台维护。
- 订阅提醒设置。

## 6. 核心功能设计

### 6.0 首次引导

新用户首次进入不能直接丢到空首页，需要先完成轻量初始化。

首次引导步骤：

1. 微信登录。
2. 设置考试日期。
3. 设置每日学习目标，默认 15 个知识点。
4. 设置提醒时间，默认 08:30，可跳过。
5. 选择资料来源：先使用内置知识库，后续再导入个人资料。
6. 生成第一天今日任务。

首次引导完成后，用户每次打开小程序直接进入今日页。

### 6.1 每日知识点

每天生成 15 个知识点，推荐比例：

| 来源 | 数量 | 说明 |
| --- | ---: | --- |
| 新知识点 | 6 | 未学习或学习次数较少的知识点 |
| 错题关联知识点 | 4 | 来自错题本的薄弱点 |
| 低掌握度知识点 | 3 | 掌握度分数较低或长期未复习 |
| 高频重点 | 2 | 系统架构师高频考点 |

知识点卡片字段：

- 标题。
- 所属章节。
- 重要程度。
- 难度。
- 3 行以内核心解释。
- 高频考法。
- 易错提醒。
- 关联题目。
- AI 追问入口。

交互建议：

- 卡片左右滑切换知识点。
- 卡片翻转查看“易错点”和“高频考法”。
- 完成后出现轻量进度动效。
- 卡片底部固定“做题验证”按钮。

未完成处理：

- 当天未完成的知识点不直接丢弃。
- 次日生成任务时，未完成项进入高优先级候选池。
- 未完成项最多连续保留 3 天，超过后回到普通复习池，避免首页压力过大。
- 今日页需要展示“昨日未完成”提示，但不制造强挫败感。

### 6.2 每点一题

每个知识点至少绑定一道题。

题目类型：

- 单选题。
- 多选题，MVP 可后置。
- 判断题。
- 案例分析小题，MVP 可后置。
- 变式题，AI 能力稳定后再做。

答题后记录：

- 用户答案。
- 是否正确。
- 答题耗时。
- 是否查看解析。
- 错误原因。
- 是否加入错题本。

错误原因枚举：

```text
概念不清
概念记混
审题错误
不会应用
知识点遗忘
```

### 6.3 AI 知识库问答

AI 问答使用 GLM-5.2 和智谱知识库检索能力。后端负责代理模型调用，前端不持有 API Key。

问答流程：

```text
用户提问 -> 后端鉴权和限流 -> 调用 GLM-5.2 知识库检索 -> 返回答案和引用 -> 保存会话记录
```

回答要求：

- 优先基于知识库回答。
- 找不到依据时明确说明“知识库中未找到直接依据”。
- 必须展示来源章节、文档或知识片段。
- 支持一键关联到知识点。
- 支持一键加入复习队列。

快捷追问：

- 讲简单点。
- 举个软考例子。
- 出一道题考我。
- 这个容易怎么考。
- 加入复习计划。

AI 回流规则：

- 从知识点进入 AI：默认关联当前知识点。
- 从错题进入 AI：默认关联当前错题和知识点。
- 普通提问若命中知识点：允许用户一键关联。
- 用户点击“加入复习”后，知识点进入复习队列，并标记为疑难点。

### 6.4 错题本

错题本是薄弱点训练中心，不只是错题列表。

分类方式：

- 按章节。
- 按知识点。
- 按错误原因。
- 按错误次数。
- 按最近错误时间。
- 按是否已掌握。

错题详情：

- 原题。
- 我的答案。
- 正确答案。
- 解析。
- 关联知识点。
- 错误原因。
- 最近错误时间。
- 重做记录。

错题操作：

- 重做。
- 查看关联知识点。
- 问 AI。
- 标记已掌握。
- 生成同类题，MVP 后置。

掌握判定：

- 用户可手动标记已掌握。
- 同一道错题连续 2 次重做正确后，系统自动标记已掌握。
- 已掌握错题默认不出现在待复习列表，但 7 天后可进入低频抽查。

### 6.5 学习计划

用户设置：

- 考试日期。
- 每日学习目标，默认 15 个知识点。
- 每日提醒时间。
- 是否开启错题优先。

计划能力：

- 自动生成每日任务。
- 根据掌握度和错题调整任务。
- 临近考试时提高错题和高频重点占比。
- 将错题、疑难点、未完成项统一纳入复习队列。

### 6.6 掌握度模型

每个用户对每个知识点都有一个掌握度分数。

初始建议：

| 行为 | 分数变化 |
| --- | ---: |
| 完成学习 | +10 |
| 答题正确 | +15 |
| 答题错误 | -15 |
| 重复答错 | -10 |
| 错题重做正确 | +20 |
| 7 天未复习 | 优先级提高 |
| 主动问 AI | 标记疑难点 |

掌握度分层：

```text
0-39   薄弱
40-69  待巩固
70-89  基本掌握
90-100 熟练
```

## 7. 页面设计方向

前端 UI 的具体视觉提示词和视频灵感分析见 `docs/05-ui-design-prompt-from-video.md`。本节保留产品层面的页面方向，后续设计和实现应优先遵循该 UI 提示词中的浅色科技感、轻玻璃卡片、空间层叠和柔和动效原则。

### 7.1 视觉风格

关键词：

```text
简洁
科技感
清爽
高完成感
低学习压力
```

建议采用双主题：

- 日间：浅色背景，适合长时间阅读。
- 夜间：深色背景，突出科技感和沉浸感。

建议色彩：

| 用途 | 色值 |
| --- | --- |
| 主色 | #3B82F6 |
| 成功 | #10B981 |
| 警示 | #F59E0B |
| 错题 | #F43F5E |
| 深色背景 | #0F172A |
| 浅色背景 | #F8FAFC |

### 7.2 关键交互

- 首页进度环随完成数量增长。
- 知识卡完成时出现轻量粒子或光效。
- 答题正确时按钮变为绿色并展示一句鼓励。
- 答题错误时不做强烈挫败反馈，直接引导看解析。
- AI 回答时支持流式输出，MVP 可先非流式。
- 错题本章节热力图展示薄弱程度。

### 7.3 页面清单

MVP 页面：

- 登录页。
- 今日首页。
- 知识点详情页。
- 答题页。
- 答题解析页。
- 错题本页。
- AI 问答页。
- 我的页。
- 学习计划设置页。

后续页面：

- 模拟考试页。
- 真题套卷页。
- 知识库管理页。
- 管理后台 Web 页。

## 8. 微信订阅消息策略

微信订阅消息不适合将 15 个知识点逐条推送给用户。产品上建议每天只推一条提醒：

```text
今日软考复习任务已生成：15 个知识点 + 15 道练习题
```

触发场景：

- 每日任务生成后提醒。
- 错题到期复习提醒。
- 考前冲刺提醒。

注意事项：

- 订阅消息需要用户授权。
- 前端通过 `wx.requestSubscribeMessage` 请求授权。
- 后端保存用户订阅状态并调用微信服务端接口发送。
- 长期订阅消息存在类目限制，MVP 按一次性订阅消息设计更稳。

## 9. MVP 范围

必须做：

- 微信登录。
- 今日 15 个知识点。
- 每点一题。
- 答题记录。
- 错题本。
- 章节掌握度。
- GLM-5.2 知识库问答。
- 学习计划设置。
- 订阅消息提醒。

暂缓：

- 多端适配。
- 复杂社区功能。
- 排行榜。
- 支付。
- 复杂管理后台权限。
- AI 自动生成大规模题库。

## 10. 验收标准

- 用户首次进入能完成登录并设置考试日期。
- 系统能生成当天 15 个知识点。
- 用户能完成知识点学习和配套答题。
- 答错题目自动进入错题本。
- 错题能按章节筛选和重做。
- AI 能基于知识库回答问题，并展示来源。
- 首页能展示今日进度、正确率、薄弱章节。
- 订阅消息授权和提醒链路可用。
