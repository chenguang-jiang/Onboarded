## 3. 视觉语言提炼

### 3.1 关键词

```text
浅色科技感
柔和渐变
轻玻璃卡片
空间层叠
安静完成感
低饱和光效
清晰学习路径
```

### 3.2 色彩

主视觉不应使用强烈荧光色，也不应变成传统深色赛博风。建议以冷白、淡蓝、浅紫、微粉作为基础，再用蓝色和绿色表达学习进度。

推荐色板：

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 页面背景 | `#F6F8FF` | 冷白底色，承接视频中的浅色空间感 |
| 背景渐变 1 | `#EAF1FF` | 淡蓝雾面光 |
| 背景渐变 2 | `#F3E8FF` | 淡紫光 |
| 背景渐变 3 | `#FFE8F1` | 极轻粉色补光 |
| 主操作色 | `#3B82F6` | 学习、继续、提交 |
| 完成色 | `#10B981` | 正确、完成、掌握 |
| 错题色 | `#F43F5E` | 错误、待复习 |
| 文本主色 | `#172033` | 保证阅读清晰 |
| 文本次色 | `#64748B` | 标签、说明、弱信息 |
| 卡片边框 | `rgba(255,255,255,0.72)` | 轻玻璃边缘 |

### 3.3 字体与排版

微信小程序优先使用系统字体：

```text
iOS: PingFang SC
Android: MiSans / system-ui
数字: DIN Alternate 或系统等宽数字回退
```

排版原则：

- 标题短，正文不超过 3-5 行。
- 学习任务用数字强化，例如 `15`、`8/15`、`42 天`。
- 卡片内留白充足，但不要做大面积营销式 hero。
- 标签小而清晰，避免装饰性胶囊铺满页面。

### 3.4 组件形态

组件风格：

- 卡片圆角 `20-28px`，比普通后台系统更柔和。
- 内层按钮圆角 `14-18px`，不能过度胶囊化。
- 卡片使用轻阴影和半透明边框。
- 背景可有极淡的光斑渐变，但不能出现明显装饰圆球。
- 输入框采用轻玻璃底，边界柔和。

核心组件：

- 今日进度环。
- 横向知识点卡组。
- 知识点详情卡。
- 答题选项卡。
- AI 消息气泡。
- 引用来源卡片。
- 错题章节热力图。
- 底部主操作栏。

### 3.5 动效与交互

动效原则：

- 动效用于确认状态，不用于炫技。
- 过渡要柔和，避免弹跳过强。
- 卡片切换像“空间层叠滑动”，不做重 3D 旋转。
- 答对题目出现绿色微光和进度增长。
- 答错题目只给明确反馈，不做刺眼红色震动。
- AI 回答使用流式出现，引用来源延迟淡入。

推荐动效：

```text
知识卡滑动：280ms ease-out
卡片进入：opacity + translateY 12px
完成反馈：进度环增长 + 轻微光晕
AI 输出：逐行流式展示
页面切换：淡入 + 轻微上移
```

## 4. Onboarding 页面设计落点

### 4.1 今日页

结构：

```text
顶部：考试倒计时、连续学习天数、AI 快问
中部：今日进度环 + 15 个知识点横向卡组
底部：当前推荐动作，例如“继续学习”“重做错题”
```

设计重点：

- 采用视频中的浅色空间背景。
- 今日知识点卡组呈现轻微层叠。
- 当前卡片突出，前后卡片露出边缘。
- 页面要一眼知道今天还剩多少。

### 4.2 知识点详情页

结构：

```text
顶部：章节、重要程度、难度标签
主体：知识点解释、考试记忆点、易错提醒
底部：做题验证、问 AI、标记已掌握
```

设计重点：

- 内容卡片像视频里的 floating card。
- 解释内容高度克制，复杂内容分区折叠。
- “考试记忆点”使用轻强调色，不使用大面积警示块。

### 4.3 答题页

结构：

```text
顶部：知识点名、当前题目进度
主体：题干、选项卡
底部：提交、下一题、问 AI 解释
```

设计重点：

- 单题沉浸，减少干扰。
- 选项卡有清晰选中态。
- 答案反馈不挤压题干位置。
- 底部主操作按钮可以使用轻微发光圆点灵感。

### 4.4 AI 问答页

结构：

```text
顶部：当前上下文，例如“正在基于 软件架构评估 回答”
主体：问答消息流
中部：引用来源卡片
底部：输入框 + 快捷追问
```

设计重点：

- 借鉴视频中的 AI 输入和搜索体验。
- 输入框常驻底部，轻玻璃质感。
- 引用来源必须可展开，不要淹没主回答。
- 快捷追问使用小型 chips：讲简单点、出题考我、举例说明。

### 4.5 错题本页

结构：

```text
顶部：搜索、错题总数、待复习数
中部：章节筛选、错误原因筛选
主体：错题卡片或章节热力图
```

设计重点：

- 借鉴视频里的 `In Progress` 页面。
- 搜索和筛选在顶部，不打断错题列表。
- 卡片网格可以承载章节，但题目列表仍用单列保证阅读。

## 5. 可直接使用的 UI 设计提示词

下面这段可以直接复制给 UI 设计工具、原型生成工具或前端实现工具使用。

```text
请为微信小程序 Onboarding 设计一套前端 UI。Onboarding 是一个软考系统架构师备考教练小程序，核心功能包括：每日 15 个知识点、每个知识点一道题、AI 基于知识库问答、错题本、章节掌握度和学习计划。

视觉风格参考一段浅色科技感 UI 视频：手机界面悬浮在冷白空间中，背景有柔和的淡蓝、浅紫、微粉渐变光，卡片像轻玻璃一样浮在界面上，信息上下分层清晰，顶部常见搜索或上下文标签，中间是内容卡片，底部是输入框或主操作按钮。整体感觉是简洁、未来感、安静、高级，但不要强烈赛博风，不要高饱和霓虹，不要复杂装饰。

请使用以下设计语言：
1. 背景使用冷白到淡蓝、浅紫、微粉的柔和渐变，主背景色接近 #F6F8FF。
2. 卡片使用轻玻璃质感：白色半透明、细边框、柔和阴影、20-28px 圆角。
3. 主操作色使用 #3B82F6，完成色使用 #10B981，错题色使用 #F43F5E，正文主色使用 #172033，辅助文字使用 #64748B。
4. 页面要简洁，信息密度适中，不做营销页，不做大段说明。用户打开首页要立刻知道今天学什么、完成多少、下一步做什么。
5. 组件需要有轻微空间层叠感，今日 15 个知识点采用横向滑动卡组，当前卡片突出，前后卡片露出边缘。
6. 动效要柔和：卡片进入使用淡入和轻微上移，知识点完成时进度环增长并出现轻微光晕，AI 回答流式出现，引用来源淡入。
7. 所有控件必须适合微信小程序，底部导航清晰，按钮可点击区域足够大，长文本不能溢出。

请设计以下核心页面：
1. 今日页：顶部展示考试倒计时、连续学习天数和 AI 快问入口；中部展示今日进度环和 15 个知识点横向卡组；底部展示下一步动作按钮。
2. 知识点详情页：展示章节、重要程度、难度、核心解释、考试记忆点、易错提醒；底部提供“做题验证”“问 AI”“标记掌握”。
3. 答题页：单题沉浸式布局，展示题干、选项、提交按钮；答题后展示正确答案、解析和“问 AI 解释这题”。
4. AI 问答页：聊天式布局，底部固定输入框，支持快捷追问 chips；AI 回答必须展示引用来源卡片。
5. 错题本页：顶部搜索和统计，中部章节筛选和错误原因筛选，主体展示错题卡片与章节薄弱热力图。
6. 我的页：展示学习计划、考试日期、每日目标、提醒设置和掌握度概览。

设计时请避免：
1. 大面积深色背景。
2. 强霓虹、高饱和渐变、过度炫光。
3. 复杂 3D 旋转或影响阅读的动效。
4. 过多装饰性胶囊标签。
5. 像后台管理系统一样冷硬，也不要像营销落地页一样夸张。

最终效果应该像一个“安静、聪明、有一点未来感的备考学习舱”：用户每天打开它，会自然进入学习、做题、问 AI 和复盘错题的节奏。
```

## 6. 前端实现补充提示词

如果后续直接让前端工程实现，可以追加下面这段：

```text
请使用微信原生小程序 + TypeScript 实现，组件优先基于 TDesign Miniprogram 扩展。样式需要抽象为 design tokens，包括颜色、圆角、阴影、间距和字体层级。页面必须适配 iPhone 和常见 Android 微信环境。动画使用小程序原生动画或 Skyline 能力，避免复杂 Web CSS 特性。所有页面要有 loading、empty、error 状态。AI 问答需要支持长文本、引用来源折叠、快捷追问和底部输入框安全区适配。今日页和答题页是核心体验，优先保证可读性、点击区域和流畅度。
```

## 7. 设计验收清单

- 首页是否一眼能看出今日任务进度。
- 卡片是否有轻空间感，但不影响阅读。
- 背景是否柔和，不抢内容。
- 主按钮是否明确。
- 答题反馈是否清晰但不刺眼。
- AI 回答是否有来源。
- 错题本是否能快速筛选章节。
- 长文本是否不溢出。
- 底部输入和按钮是否适配安全区。
- 真机滚动和滑动是否流畅。

## 8. 今日首页卡组重构提示词

下面这段用于定向重构今日首页，重点解决移动端卡片拥挤、标题折行和悬浮操作遮挡问题。

```text
你是一位精通微信小程序开发与极简科技风（Minimalist Tech Style）的 UI/UX 专家。请帮我重构小程序的“今日”首页，特别是解决目前“知识点卡组”排版拥挤、文字折行严重的问题。

全局设计规范：
1. 背景保留柔和浅紫/浅蓝渐变背景。
2. 卡片使用轻微磨砂玻璃质感的纯白或半透明面板，配合极淡弥散阴影，例如 box-shadow: 0 8px 32px rgba(0,0,0,0.05)，去掉生硬实线边框。
3. 排版使用大圆角、大面积留白和克制强调色。

顶部数据概览区：
1. 将“今天先完成 15 个关键考点”及副标题整合到一个干净毛玻璃卡片中。
2. 今日进度、连续学习、已学、已掌握、薄弱等指标使用 Grid 或横向 Flex 统一整合，避免割裂成多个面板。

核心知识点卡组：
1. 不使用普通横向 scroll-view 让多张卡片挤在一起。
2. 必须使用微信小程序 swiper，并开启 previous-margin="60rpx" 和 next-margin="60rpx"。
3. 中间激活卡片保持 transform: scale(1)、opacity: 1；两侧未激活卡片 transform: scale(0.9)、opacity: 0.6；切换使用 transition: all 0.3s ease-in-out。
4. 单张卡片头部放置“新知识/顺延/错题回顾”和“待学习/学习中/已完成”等小标签。
5. 标题必须单行显示，超出使用 text-overflow: ellipsis，绝不折行。
6. 正文最多 3 行，使用 -webkit-line-clamp: 3，行高约 1.6。
7. 卡片底部放置 100% 宽度低饱和蓝色渐变“做题验证”按钮，或优雅的下一题箭头提示。

底部操作栏：
1. 不允许挡住知识点正文。
2. 固定在屏幕底端、Tab Bar 之上，做成通栏半透明磨砂条，或融入当前激活卡片内部逻辑。
```
