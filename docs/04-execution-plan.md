# Onboarding 落地执行计划

版本：v1.0  
日期：2026-07-01  
目标：4 周完成可体验 MVP，8 周形成相对完整的备考闭环

## 1. 执行策略

先做学习闭环，再做体验增强。

优先级：

```text
项目骨架 -> 首次引导 -> 内容数据 -> 今日任务 -> 答题 -> 错题 -> AI 问答回流 -> 订阅提醒 -> 体验优化
```

不建议第一阶段投入过多时间做：

- 复杂管理后台。
- AI 自动生成大量题库。
- 支付。
- 社区。
- 多端框架。
- 微服务拆分。

## 2. 里程碑规划

### M0：项目初始化，1-2 天

目标：

- 建立前后端工程。
- 本地环境可运行。
- 数据库迁移机制可用。

任务：

- 创建微信原生小程序项目。
- 引入 TypeScript。
- 引入 TDesign Miniprogram。
- 创建 Spring Boot 3 项目。
- 配置 MySQL、Redis、Flyway。
- 创建 Docker Compose。
- 配置基础环境变量。
- 建立统一响应、异常处理、日志格式。

交付物：

- 前端可启动。
- 后端健康检查接口可访问。
- MySQL 和 Redis 可本地运行。
- 第一版数据库迁移脚本。

验收：

```text
GET /actuator/health 返回 UP
小程序首页能请求后端 health 接口
Flyway 能自动创建基础表
```

### M1：用户与内容基础，3-5 天

目标：

- 完成微信登录。
- 完成首次引导和学习设置。
- 完成章节、知识点、题库基础能力。

任务：

- 实现 `POST /api/auth/wx-login`。
- 创建用户表。
- 创建首次引导状态表。
- 实现考试日期、每日目标、提醒时间设置。
- 创建章节表。
- 创建知识点表。
- 创建题目和选项表。
- 提供章节树接口。
- 提供知识点详情接口。
- 提供题目查询接口。
- 准备第一批软考章节、知识点和题目种子数据。

交付物：

- 用户可登录。
- 前端能展示章节和知识点。
- 后端有可用样例数据。

验收：

```text
用户首次进入能完成微信登录
用户能完成考试日期、每日目标和提醒时间设置
首次引导完成后进入今日页
章节列表能正常展示
知识点详情能正常展示
每个样例知识点至少有 1 道题
```

### M2：今日任务与学习闭环，5-7 天

目标：

- 用户每天能拿到 15 个知识点。
- 能完成知识点学习和配套答题。

任务：

- 实现每日计划表。
- 实现每日计划项表。
- 实现每日任务生成服务。
- 实现复习队列 review_queue。
- 未完成任务进入次日复习候选池。
- 实现 `/api/today` 聚合接口。
- 实现知识点开始和完成接口。
- 实现题目作答接口。
- 实现答题记录表。
- 实现首页今日任务 UI。
- 实现知识点卡片 UI。
- 实现答题页和解析页。

交付物：

- 今日首页。
- 知识卡片。
- 答题流程。
- 答题记录。

验收：

```text
新用户进入能生成当天 15 个知识点
用户完成知识点后今日进度增加
昨日未完成任务能在次日重新参与排序
用户提交答案后能看到正确答案和解析
答题记录写入数据库
```

### M3：错题本与掌握度，4-6 天

目标：

- 答错自动进入错题本。
- 用户能按章节复习错题。
- 系统能计算知识点掌握度。

任务：

- 创建错题表。
- 创建掌握度表。
- 实现错题写入和更新规则。
- 实现掌握度评分规则。
- 实现错题列表接口。
- 实现错题详情接口。
- 实现错题重做接口。
- 实现标记已掌握接口。
- 实现错题本页面。
- 实现章节掌握度页面或首页摘要。

交付物：

- 错题本。
- 掌握度计算。
- 章节薄弱摘要。

验收：

```text
答错题目自动进入错题本
重复答错会增加 wrong_count
错题可按章节筛选
错题重做正确后掌握度提升
首页能展示薄弱章节
```

### M4：GLM-5.2 知识库问答，5-7 天

目标：

- 用户可以通过 AI 基于知识库提问。
- AI 回答带来源，降低幻觉风险。

任务：

- 在智谱开放平台创建知识库。
- 导入第一批软考资料。
- 配置 `GLM_API_KEY` 和 `GLM_KNOWLEDGE_ID`。
- 实现 `GlmKnowledgeClient`。
- 实现 AI 问答接口。
- 实现 AI 会话表和消息表。
- 实现用户级 AI 限流。
- 实现 AI 问答页面。
- 实现引用来源展示。
- 实现从知识点或错题跳转 AI 提问。
- 实现 AI 回答后“加入复习”的回流动作。

交付物：

- AI 问答页。
- 会话历史。
- 来源引用。
- AI 限流和错误兜底。

验收：

```text
用户能输入问题并收到 GLM-5.2 回答
回答能展示至少一个知识库来源
AI 服务超时时前端有明确提示
AI 问答记录能保存并再次打开
AI 回答可加入复习队列
```

### M5：订阅消息与提醒，2-4 天

目标：

- 支持每日任务提醒。
- 支持错题到期复习提醒。
- 支持用户拒绝订阅后的首页兜底提示。

任务：

- 申请微信订阅消息模板。
- 前端调用 `wx.requestSubscribeMessage`。
- 后端保存订阅授权记录。
- 实现微信 access_token 获取和缓存。
- 实现订阅消息发送 client。
- 每日任务生成后发送提醒。
- 错题到期后发送提醒，MVP 可后置。

交付物：

- 订阅授权。
- 每日提醒。

验收：

```text
用户可授权每日提醒
后端能成功发送订阅消息
发送失败能记录错误原因
订阅消息不阻塞每日任务生成
用户拒绝订阅后仍能在首页看到今日任务状态
```

### M6：视觉与交互打磨，3-5 天

目标：

- 让产品简洁、炫酷、有完成感。
- 保持学习路径高效。
- 按 `docs/05-ui-design-prompt-from-video.md` 统一浅色科技感和轻玻璃视觉语言。

任务：

- 完成双主题视觉规范。
- 首页进度环动效。
- 知识卡片翻转或滑动动效。
- 答题正确和错误反馈。
- 错题热力图。
- AI 快捷追问 chips。
- 页面骨架屏。
- 空状态和错误状态。

交付物：

- 完整小程序体验。
- 统一视觉风格。

验收：

```text
核心页面无明显视觉割裂
主要按钮和状态反馈清晰
弱网和空数据有友好提示
小程序真机体验不卡顿
```

## 3. 4 周 MVP 排期

### 第 1 周

重点：项目基础、登录、内容和题库。

计划：

- M0 项目初始化。
- M1 用户与内容基础。
- 准备第一批章节和题目数据。

输出：

- 可登录。
- 可看章节和知识点。
- 可取题目。

### 第 2 周

重点：今日任务、答题闭环。

计划：

- M2 今日任务与学习闭环。
- 首页、知识卡、答题页。

输出：

- 每日 15 个知识点。
- 每点一题。
- 答题记录。

### 第 3 周

重点：错题、掌握度、AI。

计划：

- M3 错题本与掌握度。
- M4 GLM-5.2 知识库问答的后端和基础 UI。

输出：

- 错题本。
- 章节薄弱度。
- AI 问答。

### 第 4 周

重点：提醒、体验、上线准备。

计划：

- M5 订阅消息。
- M6 视觉与交互打磨。
- 部署测试环境。
- 真机测试。

输出：

- 可体验 MVP。
- 部署环境。
- 验收报告。

## 4. 8 周增强路线

当前实现状态（2026-07-03）：

- M4 已完成基础闭环：GLM-5.2 mock/real switch、会话与消息持久化、来源展示、AI 页面、上下文跳转、AI 回答加入复习队列。
- M5 已完成授权记录闭环：Profile 页发起订阅授权，后端保存接受/拒绝状态，MySQL 持久化。真实订阅消息发送仍需微信审核通过的模板 ID、生产 AppSecret 和 access_token 发送 client。
- M6 已完成核心视觉打磨：Today 画廊卡组、Answer 翻转知识卡、AI 知识库式回答、Wrongbook 热力图和错误原因标签。
- 第 7 周增强已完成入口级能力：错题可一键带上下文问 AI，AI prompt 会要求解释错题并生成变式题；后续可再落成结构化变式题入库。

第 5 周：

- 管理后台 MVP。
- 章节、知识点、题目维护。
- Excel/Markdown 导入。

第 6 周：

- 模拟考试。
- 真题套卷。
- 考试计时和分数统计。

第 7 周：

- AI 解释错题。
- AI 生成变式题。
- 高频问题沉淀 FAQ。

第 8 周：

- 个性化推荐增强。
- 学习报告。
- 考前冲刺计划。
- 成本和监控面板。

## 5. 人员分工建议

单人全栈模式：

```text
第 1 优先级：后端数据模型和核心接口
第 2 优先级：小程序核心页面
第 3 优先级：AI 问答和订阅消息
第 4 优先级：动效和体验打磨
```

两人模式：

- A：后端、数据库、AI、微信服务端。
- B：小程序页面、交互、视觉、接口联调。

三人模式：

- A：后端业务和数据库。
- B：小程序前端。
- C：内容数据、题库、AI 知识库、测试。

## 6. 数据准备计划

第一批内容建议：

- 10-15 个一级章节。
- 每章 10-20 个知识点。
- 每个知识点至少 1 道题。
- 高频章节优先。

优先章节：

- 软件架构基础。
- 软件架构风格。
- 软件质量属性。
- 软件架构评估。
- 分布式系统。
- 数据库系统。
- 信息安全。
- 系统分析与设计。
- 项目管理。
- 案例分析常见题型。

数据格式建议：

```text
chapters.xlsx
knowledge_points.xlsx
questions.xlsx
```

后续导入到 MySQL，同时把资料文档导入智谱知识库。

## 7. 测试计划

### 7.1 后端测试

必须覆盖：

- 每日任务生成。
- 题目判题。
- 错题写入。
- 掌握度计算。
- AI 调用降级。
- 订阅消息发送失败。

### 7.2 前端测试

必须覆盖：

- 登录流程。
- 今日任务空态和有数据状态。
- 知识卡滑动和完成。
- 答题正确、错误、解析。
- 错题筛选。
- AI 问答加载、成功、失败。
- 订阅授权成功和拒绝。

### 7.3 真机测试

设备：

- iPhone。
- Android。
- 微信开发者工具。

关注：

- 页面滚动性能。
- 动效是否卡顿。
- 长文本是否溢出。
- AI 回答较长时是否可读。
- 弱网下是否有明确反馈。

上线前检查：

- 隐私政策。
- 用户协议。
- 微信登录配置。
- 订阅消息模板。
- GLM API Key。
- 知识库 ID。
- 生产数据库备份策略。
- 日志和错误告警。

## 9. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 知识库检索效果不稳定 | AI 回答质量波动 | 优化资料切分、Prompt、来源展示 |
| 题库准备慢 | MVP 学习闭环薄弱 | 先覆盖高频章节，每点至少 1 题 |
| 小程序订阅消息限制 | 提醒能力弱 | 只做任务生成提醒，前端加强首页回访 |
| 动效影响性能 | 学习体验变差 | 动效轻量化，真机测试后保留必要部分 |
| AI 成本不可控 | 运营成本上升 | 用户级限流、缓存、统计 token 用量 |

## 10. MVP 完成定义

满足以下条件视为 MVP 完成：

- 用户可以登录并设置考试日期。
- 系统每天生成 15 个知识点。
- 每个知识点可以进入配套题。
- 答题后能看到解析。
- 错题自动进入错题本。
- 错题可按章节筛选并重做。
- AI 能基于知识库回答问题并展示来源。
- 首页能展示今日进度和薄弱章节。
- 订阅消息提醒链路可用。
- 小程序真机核心路径可顺畅完成。

建议下一步先执行 M0 和 M1：

4. 在智谱控制台创建知识库。
5. 准备第一批导入资料。
